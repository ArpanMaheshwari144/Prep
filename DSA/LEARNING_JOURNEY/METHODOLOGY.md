# METHODOLOGY — How I Learn DSA (For Me & Claude)

**Last updated:** 2026-04-19 (after reset session)

---

## 🧠 MY LEARNING STYLE (identified)

1. **Visualization is AUTOMATIC** — no effort. Say "restaurant" → image forms.
2. **Minimalistic** — all black, no colors, no fancy details. Only essential action.
3. **Spatial fluid** — can project to USA, Europe, any scenario.
4. **Systems thinking** — I see flows (string → server → response) naturally.
5. **Concept-first, not problem-first** — 20 problems grinding useless. Concept deeply → problems auto.
6. **Note-taking kinesthetic** — read → rewrite in my own words → locked permanently.
7. **2 problems per concept = enough.** Lock via variety, not volume.

---

## 🎯 TEACHING FLOW (per concept)

### **Step 1 — ANALOGY** (5 min)
- Real-world scene, simple, minimalistic.
- My brain auto-visualizes. No fancy colors needed.
- Examples:
  - Recursion = Russian doll / box inside box / CEO delegation
  - Stack = plate tower
  - Queue = line at counter
  - Tree = family tree
  - Graph = city with roads

### **Step 2 — CORE CONCEPT** (10 min)
- Structure + action only.
- "Iss pattern ka dil" — 1-line essence.
- NO walls of theory.

### **Step 3 — DRY RUN** (15 min)
- Visual trace with actual values.
- State changes step-by-step.
- No pseudocode (if concept still new).

### **Step 4 — PSEUDOCODE** (OPTIONAL, 5 min)
- ONLY if concept clear in my head.
- Pseudocode reinforces — I translate to C++ myself.
- Skip if I'm still confused on concept.

### **Step 5 — I CODE** (20 min)
- No skeleton from Claude.
- Stuck → Claude shows dry run of my code (Socratic).
- I figure out the bug myself.

### **Step 6 — 1-2 MORE PROBLEMS** (30 min)
- Variety of same pattern.
- Lock concept via different flavors.

### **Step 7 — SELF-NOTES** (5 min)
- Rewrite key insight in my own words.
- Daily scan — repeat = permanent memory.

---

## ⚡ SACRED RULES (Claude never violates)

## 🚀 INDEPENDENCE PROGRESSION

### **Phase 1 — Scaffolding (current)**
- Claude: concept + analogy + dry run + problem pick
- Me: code only
- Today's problems — Fibonacci, Pow etc

### **Phase 2 — Light scaffolding (2-3 weeks)**
- Claude: problem category + 1 hint
- Me: approach + code

### **Phase 3 — Independent (1-2 months)**
- Claude: problem link
- Me: pattern identify + solve

### **Phase 4 — Interview ready (2 months)**
- Novel problems — 2-3 min to identify pattern
- Code confidently
- Defend approach under interviewer pressure
