# "Why JP Morgan?" — answer (honest, interview-ready) [behavioral #1 trap]

> money = engine (ANDAR rakho), answer = fit + interest + growth (BAHAR bolo).
> "paisa kamane aaya hu" bolna = INSTANT FAIL (no thought/fit -> "+1 rupee pe chhod dega").

## THE ANSWER (bolna — ~30-45 sec, 3 reasons)
```
   1. DOMAIN — "I genuinely enjoy building reliable money-systems. I've fixed payment issues in prod and
      recently built a payment microservice with SAGA + consistency handling — the correctness / no-double-spend
      problems interest me. JP is payments and money at its core."

   2. FIT — "My background is Java/Spring backend + production reliability (4 years, 650+ tickets).
      That's JP's core stack and the day-to-day — maintaining and extending large systems. I can contribute early."

   3. SCALE + GROWTH — "JP moves ~$10T/day. Working on systems at that scale, at a top-tier firm,
      is where I want to grow as an engineer."
```

## SHORT version (agar crisp maangein)
```
   "Java/Spring backend + reliability is my strength, payments/consistency genuinely interests me,
    and JP is where I can do that at real scale."
```

## KYA NAHI BOLNA
```
   - "paisa / package / 40 LPA"      -> instant fail (no fit/thought)
   - generic "bada brand / MNC"       -> sabko pata, research nahi dikhta
   - current-job pe blame / complaint -> negative signal
```

## DELIVERY (behavioral = delivery, likhna nahi)
```
   - CONFIDENT + SPECIFIC -> project (SAGA payment-svc) + 650 tickets se back karo, hawa nahi.
   - unke DOMAIN se connect -> payments/money/reliability (generic nahi).
   - BOLNE ki practice -> loud, crisp, 30-45 sec. likh ke chhodna nahi.
```

---
> yaad: sab SACH hai (payment-project + Java + reliability + scale-interest) — fake nahi, bas money andar.
