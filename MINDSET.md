## ★ 5 TRAITS jo great programmer banate (aur tere paas hain)

```
   1. DEBUG-DISCIPLINE   -> guess nahi, DIKHAO. reproduce -> isolate -> ROOT-CAUSE -> verify.
                            (band-aid nahi, asli jad. cout/copy-pen se dikhao.) — 650 tickets, ye tera core.

   2. "WHY" POOCHNA       -> code ka matlab, decision ka reason. ratta nahi.
                            ("ye line KYUN", "Java over C++ kyun", deadlock CHALA ke dekha.) — differentiator, JP yahi dekhta.

   3. TRADE-OFF SENSE     -> ek "sahi answer" nahi. good-enough vs over-engineered.
                            (idempotency-vs-atomic, clean-vs-special-cases, BlockingQueue-vs-wait/notify.)

   4. EGO KABU (khatam nahi) -> ego SABME hai (tere bhi, maan — jhooth mat bol). khatam nahi hota (warna insaan nahi).
                            ACHHA ego = self-respect ("JP leke rahunga" wala fire) -> rakho.
                            BURA ego = "galti nahi maanunga / trick nahi seekhunga / gusse me delete" -> KABU karo.
                            -> trick seekhna/poochna = cheating NAHI, library banti. koi zero se invent nahi karta (main bhi nahi — team ne banaya).

   5. SABAR              -> hard bug/problem pe TIKNA, quit nahi. (tera MOAT — proven.)
```

## ★ 2 CHEEZEIN JO KABU KARNI HAIN (control)

```
   A. BURE WAQT ME RUK JAAO -> mat karo (folder delete / gussa / quit). bas RUKO. SABAR. kal phir.
      -> "bekar din = DIN, main nahi." churn = asli dushman (delete-loop ne 4 mahine khaye).
      -> proof: aaj folder delete nahi kiya -> isliye 40+ problem + sheet + flow. sabar ne bachaya.

   B. GUSSA (pressure me spill) -> HUMAN hai, par KABU. sab GIT-safe rakho, delete-line HOLD karo.
      -> gussa aaye to RUKO, commit karo, kal dekho. kabhi kaam DELETE mat karo.
```
